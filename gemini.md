# Project Constitution (gemini.md)

## Data Schemas
- **Input:** Данные, тексты и требования, которые предоставляет пользователь в чате. Скилл UI/UX дизайна (предоставит пользователь).
- **Output:** Исходный код потрясающего, быстрого, неглючного и адаптивного лендинга (HTML, CSS, JS). Максимальный упор на визуальную составляющую и сохранение текущего стиля с его последовательным улучшением.

## Behavioral Rules
- Priority: Высшее качество дизайна (UI/UX), отзывчивость на мобильных и надежность.
- ВСЕ КОМАНДЫ ПИШУТСЯ ДЛЯ ПОЛЬЗОВАТЕЛЯ (я пишу, пользователь запускает сам в PowerShell). 
- Всегда общаться на русском языке.
- **Планирование перед кодом:** Перед любой задачей расписывать краткий план/тезисы (как я понял задачу), и только после подтверждения приступать к верстке/кодингу.
- **Запрет на удаление данных:** Ни в коем случае не удалять существующий контент, код или элементы без явного на то приказа пользователя. Если нужно удалить — сначала спросить.

## Architectural Invariants
- Проект сфокусирован на фронтенд-разработке (Landing Page).
- Используется визуальный редактор кода с настроенным "live reload" (горячей перезагрузкой) для мгновенного отображения изменений и редактирования шрифтов.
- Изменения ядра строго по согласованию.
